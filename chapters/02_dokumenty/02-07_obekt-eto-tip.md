## § 2.7. Объект = тип: создав справочник, ты создал не только список

### Открытие

Вернитесь к моменту, когда мы создавали справочник «Товары» в §1.7. После его создания мы открыли палитру типов — и там появился `СправочникСсылка.Товары`. Это не случайность. Это фундаментальное свойство дерева метаданных.

### Что на самом деле происходит при создании объекта

Когда вы создаёте справочник «Товары» в Конфигураторе, вы делаете сразу две вещи:

**Первое.** Создаёте список — место для хранения записей о товарах в базе данных.

**Второе.** Создаёте новый тип данных — `СправочникСсылка.Товары`. Теперь любой реквизит любого объекта в конфигурации может иметь этот тип и ссылаться на товары.

Это второе — неочевидно и важно.

> Создав объект метаданных, вы добавляете в конфигурацию новый тип данных. Дерево метаданных — это одновременно структура хранения данных и система типов конфигурации.

### Два слоя одного действия

```
Действие: добавить справочник «Товары»
│
├── Слой данных:
│   └── В базе появляется таблица для хранения записей о товарах
│
└── Слой типов:
    └── В палитре типов появляется «СправочникСсылка.Товары»
        └── Теперь другие объекты могут на него ссылаться
```

Примитивные типы — Строка, Число, Дата, Булево — существовали до вас: они встроены в платформу (вспомните таблицу из §1.2). Ссылочный тип `СправочникСсылка.Товары` появился только потому, что вы создали этот справочник. Уберёте справочник — тип исчезнет.

### Каждая ветка дерева — новый тип

Это работает не только для справочников. Посмотрите в палитру типов — там есть `ДокументСсылка.ПоступлениеТоваров`. Значит, и создание документа добавляет тип. Любой объект дерева — новый тип.

```
Дерево метаданных          →   Система типов
│                               │
├── Справочники                 ├── СправочникСсылка...
│   ├── Товары                  │   ├── .Товары
│   └── Контрагенты             │   └── .Контрагенты
│                               │
└── Документы                   └── ДокументСсылка...
    └── ПоступлениеТоваров          └── .ПоступлениеТоваров
```

### Почему это важно

Когда вы проектируете конфигурацию, каждое решение о добавлении объекта — это одновременно решение о добавлении типа. Хорошо спроектированная конфигурация — это система, где типы объектов образуют связную сеть: документы ссылаются на справочники, регистры связаны с документами-регистраторами.

К концу курса эта сеть будет понятна вам как целое.

### Главное

Каждый объект в дереве метаданных — это одновременно структура хранения и новый тип данных. Примитивные типы встроены в платформу и существуют без вас. Ссылочные типы вы создаёте сами, добавляя объекты в дерево. Убрать объект — значит убрать тип. Дерево метаданных и система типов — это одно и то же.

### Контрольные вопросы

1. Что именно создаёт разработчик, когда добавляет справочник в дерево метаданных?
2. Откуда берутся примитивные типы — кто их создаёт?
3. Что произойдёт с реквизитами, которые ссылались на справочник, если этот справочник удалить?
4. Может ли документ ссылаться на другой документ? Какой тип для этого нужен?
5. Как связаны дерево метаданных и палитра типов?

### Упражнения

**Найди в Конфигураторе.** Создайте новый тестовый справочник с любым именем (например, «ТестовыйСписок»). Не добавляйте реквизитов. Нажмите ОК. Теперь откройте палитру типов в любом другом объекте. Нашли новый тип? Удалите тестовый справочник. Снова откройте палитру — тип исчез?

**Посмотри и ответь.** Откройте палитру типов в своей конфигурации. Подсчитайте, сколько ссылочных типов там сейчас. Сколько из них — справочники, сколько — документы? Что это говорит о текущем состоянии конфигурации?

**Спроектируй.** У фирмы «Канцтовары» скоро появится документ «РеализацияТоваров». Опишите: какие новые типы появятся в палитре после его создания? Как это повлияет на возможности других объектов конфигурации?

### Ответы и указания

*Вопрос 3.* Если удалить справочник, на который ссылаются реквизиты других объектов, система предупредит об ошибке или откажется удалять. Нельзя удалить тип, который используется — сначала нужно убрать все ссылки на него.

*Упражнение «Спроектируй».* После создания документа «РеализацияТоваров» в палитре появится `ДокументСсылка.РеализацияТоваров`. Теперь другие объекты смогут ссылаться на этот документ. Например, документ «ВозвратОтКлиента» сможет ссылаться на исходную реализацию — чтобы было понятно, какая именно продажа возвращается.
