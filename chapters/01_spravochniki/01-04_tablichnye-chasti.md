## § 1.4. Табличные части — список строк внутри элемента

### Открытие

Реквизит хранит одно значение. Иногда этого мало. У одного контрагента может быть несколько контактных лиц: директор, бухгалтер, менеджер по закупкам. Добавить десять отдельных реквизитов «Контакт1», «Контакт2»... — решение негибкое и некрасивое. Для таких случаев есть табличная часть.

### Проблема одного значения

Справочник «Контрагенты». Реквизит «Телефон» — один. Но «ООО Ромашка» — крупная фирма, у неё три контактных лица, у каждого свой телефон и должность. Как хранить всё это?

Первый вариант: добавить реквизиты «Телефон2», «Телефон3», «КонтактноеЛицо1», «КонтактноеЛицо2»... Проблема: заранее не знаем, сколько контактов будет у каждого контрагента. У одного — один, у другого — десять.

Второй вариант — табличная часть. Это таблица внутри элемента справочника. Строк может быть сколько угодно.

> Табличная часть — это таблица, вложенная в объект. Каждая строка таблицы содержит свой набор реквизитов. Количество строк не ограничено заранее.

### Как выглядит табличная часть

```
Справочник «Контрагенты»
│
├── Реквизиты:
│   ├── Наименование: «ООО Ромашка»
│   ├── ИНН: «7701234567»
│   └── Телефон: «+7 495 000-00-00»
│
└── Табличная часть «Контактные лица»:
    ├── Строка 1: Имя «Иванов А.В.»,  Должность «Директор»,   Телефон «...»
    ├── Строка 2: Имя «Петрова С.И.», Должность «Бухгалтер», Телефон «...»
    └── Строка 3: Имя «Сидоров К.Р.», Должность «Менеджер»,  Телефон «...»
```

Табличная часть — это вложенная таблица, строки которой имеют заданный набор колонок (реквизитов табличной части).

### Табличная часть — не всегда нужна

Большинство небольших справочников обходятся без табличных частей. Справочник «Склады» из трёх позиций, справочник «Валюты» — им достаточно реквизитов.

Табличная часть нужна, когда у одного элемента справочника есть повторяющийся набор данных с переменным числом строк.

### Где смотреть в Конфигураторе

```
Справочник → вкладка «Данные»
│
├── Реквизиты
│   └── (список реквизитов)
│
└── Табличные части
    └── КонтактныеЛица
        ├── Имя (Строка)
        ├── Должность (Строка)
        └── Телефон (Строка)
```

### Главное

Табличная часть — это таблица внутри элемента справочника. Она нужна, когда у одного элемента может быть произвольное количество однотипных строк с данными. Каждая строка имеет собственные реквизиты — это колонки таблицы. Без табличной части такие данные пришлось бы хранить в отдельных неудобных конструкциях или мириться с жёстко заданным числом полей.

### Контрольные вопросы

1. Чем табличная часть отличается от обычного реквизита?
2. Можно ли у одного справочника создать несколько табличных частей?
3. Приведите пример, когда табличная часть в справочнике нужна, и пример, когда она избыточна.
4. Есть ли у строки табличной части тип, как у реквизита?
5. Почему нельзя просто добавить много реквизитов вместо табличной части?

### Упражнения

**Найди в Конфигураторе.** Откройте несколько справочников из демо-базы. Найдите хотя бы один, у которого есть табличная часть. Выпишите: как называется табличная часть? Какие реквизиты в ней есть?

**Посмотри и ответь.** Откройте справочник, в котором есть табличная часть. Посмотрите на её реквизиты. Ответьте: что за информация хранится в этой таблице? Почему разработчик не сделал эти данные обычными реквизитами справочника?

**Спроектируй.** Нарисуйте структуру справочника «Поставщики» для фирмы по ремонту квартир. Включите реквизиты и хотя бы одну табличную часть. Объясните, зачем нужна табличная часть в вашем варианте.

### Ответы и указания

*Вопрос 2.* Да, у одного справочника может быть несколько табличных частей. Например, у контрагента — «Контактные лица» и «Банковские счета».

*Вопрос 5.* Если заранее неизвестно, сколько строк понадобится, фиксированное число реквизитов не работает. У одного контрагента один контакт, у другого — двадцать. Двадцать реквизитов будут пустыми у большинства записей — это неэффективно и неудобно.
