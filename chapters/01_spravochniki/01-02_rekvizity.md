## § 1.2. Реквизиты — свойства элемента

### Открытие

Справочник — это список. Но каждый элемент списка — не просто строка с названием. «Ручка шариковая» — это имя. А ещё у неё есть цена, артикул, единица измерения. Эти свойства называются реквизитами.

### Проблема без реквизитов

Представьте справочник «Товары» в виде простого листа бумаги: одна колонка, одни названия. Менеджер смотрит на «Ручка шариковая» и не знает цену. Бухгалтер не знает артикул. Кладовщик не знает, в чём измеряется товар — в штуках или упаковках.

Одного названия недостаточно. Каждый элемент справочника должен нести с собой набор свойств — всё, что бизнесу нужно знать об этой сущности.

> Реквизит — это свойство элемента справочника. Каждый реквизит хранит одно значение определённого вида.

В языке дерева реквизиты — **прилагательные**: они описывают свойства существительного-справочника. Справочник «Товары» — существительное. Реквизиты «Цена», «Артикул», «Единица» — прилагательные, которые говорят, *какой* этот товар.

### Что такое тип реквизита

У каждого реквизита есть тип — он говорит системе, что именно в нём хранится. Четыре основных типа:

| Тип | Что хранит | Пример |
|-----|-----------|--------|
| Строка | Текст любой длины | «Ручка шариковая синяя» |
| Число | Целое или дробное число | 12.50, 100, 0 |
| Дата | Дата и/или время | 15.03.2025 |
| Булево | Да или Нет | Да (товар активен) |

Эти четыре типа называют **примитивными**: они встроены в платформу и хранят значение непосредственно, без ссылок на другие объекты. Про это — подробнее в §1.7.

Тип определяет, что можно делать с реквизитом. Два числа можно сложить — система посчитает итог. Две строки нельзя арифметически сложить: «Ручка» + «Бумага» не даёт числового смысла, их можно только склеить (объединить). Система не позволит смешивать разные типы в вычислениях — это защита от ошибок.

В выноске — строгая формулировка:

> Тип данных определяет множество допустимых значений реквизита и операций над ним.

### Реквизиты справочника «Товары»

Для фирмы «Канцтовары» справочник «Товары» может выглядеть так:

```
Справочник «Товары»
├── Наименование (Строка)     ← обязательно, есть в каждом справочнике
├── Артикул (Строка)
├── Цена (Число)
└── Единица (Строка)
```

Реквизит «Наименование» есть в каждом справочнике автоматически — это имя элемента. Остальные реквизиты добавляет разработчик в зависимости от задачи.

### Где смотреть в Конфигураторе

Откройте любой справочник двойным щелчком. Перейдите на вкладку «Данные». Там — список реквизитов с указанием типа для каждого.

```
Справочник → вкладка «Данные» → раздел «Реквизиты»
│
├── Наименование   Строка   длина 150
├── Артикул        Строка   длина 25
├── Цена           Число    точность 2
└── Единица        Строка   длина 10
```

### Главное

Реквизит — это свойство элемента справочника: одна колонка в таблице, одно поле в карточке. У каждого реквизита есть тип — Строка, Число, Дата или Булево. Тип определяет, что можно хранить и что делать с этим значением. Набор реквизитов делает элемент справочника полноценным описанием бизнес-сущности, а не просто строкой с именем.

### Контрольные вопросы

1. Что такое реквизит? Чем он отличается от названия элемента справочника?
2. Какой тип подойдёт для хранения телефонного номера? Почему не Число?
3. Какой тип подойдёт для реквизита «Дата рождения сотрудника»?
4. Почему нельзя смешивать типы при вычислениях?
5. Придумайте три реквизита для справочника «Клиенты» цветочной фирмы и укажите тип каждого.

### Упражнения

**Найди в Конфигураторе.** Откройте справочник «Контрагенты» (или любой другой, если этого нет в вашей базе). Перейдите на вкладку «Данные». Выпишите все реквизиты и их типы.

**Посмотри и ответь.** Посмотрите на реквизит «Наименование». Его тип — Строка. Почему здесь не подходит тип Число? Что изменится в работе системы, если у справочника «Товары» вообще не будет реквизита «Цена» — а цену будут писать в названии товара?

**Спроектируй.** Опишите реквизиты для справочника «Сотрудники» ремонтной фирмы. Укажите имя каждого реквизита и его тип. Обоснуйте выбор типа для каждого.

### Ответы и указания

*Вопрос 2.* Телефонный номер — Строка, не Число. Причина: номер начинается с «+7» или «8», содержит скобки и тире. Число не может хранить текстовые символы. Кроме того, с телефонным номером не нужно производить арифметические операции — складывать номера не нужно.

*Вопрос 5.* Возможные варианты: «Имя» (Строка), «Телефон» (Строка), «Дата рождения» (Дата). Если вы добавили «Постоянный клиент» — подойдёт Булево (Да/Нет).

### Мостик
