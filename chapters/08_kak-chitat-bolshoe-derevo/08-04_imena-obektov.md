## § 8.4. Как читать имена объектов

### Открытие

Вы открываете незнакомую конфигурацию и видите имена: «НоменклатураДопСведения», «РегламентированныеОтчётыСПодписями», «ЗаявкиНаПроведениеПлатежейПодчинённость». Прежде чем открывать объект, попробуйте прочесть его имя как зашифрованное сообщение — в нём чаще всего уже есть ответ.

Имена объектов в 1С пишутся без пробелов, с заглавной буквы каждого слова (CamelCase по-русски). Это соглашение позволяет читать имя как фразу, если правильно разбить его на слова.

### Структура имени: основа + уточнения

Большинство имён строятся по схеме:

```
[Что] + [О чём/Для чего] + [Уточнение]

НоменклатураДопСведения
└── Номенклатура + ДопСведения
    «дополнительные сведения о номенклатуре»

РасчётыСПоставщиками
└── Расчёты + С + Поставщиками
    «расчёты с поставщиками»

ЗаявкиНаПроведениеПлатежейПодчинённость
└── Заявки + НаПроведение + Платежей + Подчинённость
    «подчинённость заявок на проведение платежей»
    (регулирует иерархию или связь между заявками)
```

Разбивайте имя на смысловые блоки. Если имя длинное — читайте слева направо: первое слово обычно называет тип сущности, остальные уточняют.

### Типичные суффиксы и что они означают

Суффиксы в именах объектов — устойчивые сигналы:

```
...ДопСведения         — дополнительные сведения (обычно регистр сведений
                          для хранения характеристик основного объекта)
...Расширенный         — расширенная версия базового объекта
...Подчинённость       — описывает иерархию или подчинение
...ПоОрганизации       — данные в разрезе организации
...КОплате             — суммы, ожидающие оплаты
...Остатки / ...Оборот — виртуальные таблицы регистра накопления
...Срезы / ...Срез     — виртуальная таблица регистра сведений (актуальное значение)
```

Если встретите «НоменклатураДопСведения» — это, скорее всего, регистр сведений, который хранит дополнительные свойства номенклатуры: те самые «прилагательные пользователя» из §7.4.

### Типичные префиксы и что они означают

Некоторые конфигурации используют префиксы для обозначения подсистемы:

```
БП_    — объект из конфигурации «Бухгалтерия предприятия»
УТ_    — из «Управление торговлей»
ЗК_    — «Зарплата и кадры»
УПП_   — «Управление производственным предприятием»
```

Такие префиксы появляются при объединении нескольких конфигураций в одну или при совместном использовании. Если видите префикс — перед вами объект из конкретной прикладной области.

### Слова-маркеры назначения

Некоторые слова в имени почти однозначно говорят о назначении:

```
Регистратор     — объект управляет движениями в регистре
Помощник        — вспомогательный объект для интерфейса
Буфер           — временное хранилище (часто технический объект)
Счётчик         — хранит числовое значение-счётчик
Настройки       — параметры конфигурации или пользователя
Журнал          — объект-журнал для просмотра
История         — хранит историю изменений
Версия          — версионирование объектов
```

Если имя содержит «Буфер» или «Вспомогательный» — скорее всего, это технический объект, не имеющий прямого бизнес-смысла. При первом чтении такие объекты можно пропустить.

### Когда имя совсем не помогает

Иногда имена — это аббревиатуры, исторические артефакты или просто плохо придуманные названия. «МТД», «ОперацияСВыбором», «ДокументОснование» — по ним невозможно понять назначение только из имени.

В таком случае алгоритм простой:

```
1. Посмотрите тип объекта (справочник? документ? регистр?)
   → это скажет, что объект в принципе умеет делать
2. Раскройте структуру (реквизиты, измерения)
   → имена полей часто говорят больше, чем имя объекта
3. Посмотрите связи (кто ссылается, кто пишет)
   → по контексту понять назначение легче, чем по имени
```

Подробный алгоритм — в §8.7. Сейчас запомните главное: непонятное имя — не тупик, это повод перейти к структуре.

### Главное

Имена объектов в 1С — зашифрованные фразы. Разбивайте имя на смысловые блоки: первое слово называет сущность, остальные уточняют. Суффиксы («ДопСведения», «Остатки», «Подчинённость») — устойчивые сигналы о назначении. Слова «Буфер», «Помощник», «Вспомогательный» — признак технического объекта, который можно пропустить при первом чтении. Если имя совсем непонятно — смотрите на структуру и связи объекта, а не пытайтесь угадать по имени.

### Контрольные вопросы

1. По какому принципу строятся имена объектов в 1С — что стоит на первом месте?
2. Что означает суффикс «ДопСведения» в имени объекта?
3. Вы видите объект «ЗарплатаКВыплатеПоОрганизациям». Что из имени можно понять о его назначении?
4. Для чего нужны префиксы типа «УТ_» или «БП_»?
5. Вы встретили объект «Обр_ПомощникПодбораТоваров». Стоит ли его подробно читать при первом знакомстве с конфигурацией?

### Упражнения

**Найди в Конфигураторе.** Откройте список регистров сведений в демо-базе. Найдите три регистра с суффиксом «ДопСведения» или «Характеристики» в имени. Для каждого проверьте: что за измерения у него? На какой справочник ссылается основное измерение? Это подтверждает предположение о назначении из имени?

**Посмотри и ответь.** Найдите в дереве объект с длинным составным именем (5 слов и более). Разбейте имя на смысловые блоки и сформулируйте предположение о назначении. Потом откройте объект и проверьте: насколько точным оказалось предположение? Что в структуре объекта вас удивило или подтвердило ожидание?

**Спроектируй.** Вам нужно добавить в конфигурацию следующие объекты. Придумайте им имена по соглашениям 1С: (1) справочник для хранения складов организации; (2) документ для оформления внутреннего перемещения товара между складами; (3) регистр сведений для хранения дополнительных характеристик поставщиков; (4) отчёт по задолженности покупателей в разрезе менеджеров.

### Ответы и указания

*Вопрос 3.* «Зарплата + КВыплате + ПоОрганизациям»: это, скорее всего, регистр накопления или сведений, который хранит суммы зарплаты, ожидающей выплаты, в разрезе организаций. Суффикс «КВыплате» — деньги ещё не выплачены, «ПоОрганизациям» — аналитика по организации.

*Вопрос 5.* Нет, не стоит при первом чтении. Префикс «Обр_» — сокращение от «Обработка», это вспомогательный интерфейсный инструмент. Слово «Помощник» подтверждает: объект для пользовательского интерфейса, не несёт самостоятельного учётного смысла.

*Упражнение «Спроектируй».* Варианты: (1) «Склады»; (2) «ПеремещениеТоваров»; (3) «КонтрагентыДопСведения» или «ХарактеристикиПоставщиков»; (4) «ДолгПокупателейПоМенеджерам».

### Мостик

Теперь вы можете читать имена и строить гипотезы ещё до открытия объекта. Следующий шаг — научиться фиксировать то, что вы читаете: строить карту конфигурации, которая помогает не забыть связи между объектами.
