## § 7.2. Планы счетов и регистры бухгалтерии

### Открытие

В Модуле 4 мы познакомились с регистром накопления: он суммирует приходы и расходы по каждому сочетанию измерений и даёт остаток. Это хорошо работает для складского учёта: «сколько товара X на складе Y».

Бухгалтерия устроена иначе. Там каждая операция записывается сразу в двух местах: одна сторона уменьшается, другая — увеличивается. Эта логика «двойной записи» требует другого инструмента.

Откройте Конфигуратор. Найдите в дереве ветку **«Планы счетов»**, а рядом — **«Регистры бухгалтерии»**. Они всегда идут парой.

### План счетов — это справочник адресов учёта

В бухгалтерии каждая операция «попадает» на определённый счёт. Счёт — это адрес, куда относится та или иная сумма. Счёт 41 «Товары» — здесь учитываются товары на складе. Счёт 60 «Расчёты с поставщиками» — здесь учитывается задолженность перед поставщиками.

План счетов — это и есть список всех таких адресов конфигурации. В дереве метаданных это специальный объект, который внешне похож на справочник, но с особыми свойствами.

```
План счетов «Основной»
├── 10  Материалы
├── 41  Товары
├── 51  Расчётные счета
├── 60  Расчёты с поставщиками
├── 62  Расчёты с покупателями
└── ...
```

Каждый счёт имеет характеристики: активный (только остаток в плюс), пассивный (только остаток в минус) или активно-пассивный (остаток бывает в любую сторону). На счёте 41 «Товары» остаток всегда активный — товары не бывают «отрицательными» в бухгалтерском смысле.

### Субконто — дополнительная аналитика счёта

Знать, что «на счёте 41 лежит 150 000 рублей», — недостаточно. Бухгалтер хочет знать: *какого* товара на сколько? На *каком* складе?

Для этого к счёту привязываются **субконто** — дополнительные аналитические разрезы. Субконто «Номенклатура» на счёте 41 означает: учёт ведётся не просто по счёту, а по каждой номенклатурной позиции отдельно. Добавляем субконто «Склад» — и теперь учёт ведётся по каждой позиции на каждом складе.

```
Счёт 41 «Товары»
├── Субконто 1: Номенклатура  (→ Справочник «Номенклатура»)
└── Субконто 2: Склад         (→ Справочник «Склады»)

Результат: регистр знает не просто «сколько на счёте 41»,
а «сколько товара X на складе Y на счёте 41».
```

Субконто — это и есть дополнительная аналитика, которая «уточняет» счёт. Без субконто счёт — просто сумма. С субконто — детализированный учёт по нужным разрезам.

### Регистр бухгалтерии — куда попадают проводки

Регистр бухгалтерии — это таблица, в которую документы пишут бухгалтерские проводки. Каждая проводка — это запись вида:

```
Дата: 15.03.2024
Документ: ПоступлениеТоваров №5
Дебет:  счёт 41 «Товары»       субконто: [Карандаш HB] [Склад №1]
Кредит: счёт 60 «Поставщики»   субконто: [ООО «Канцопт»]
Сумма:  12 000 руб.
```

Логика двойной записи: каждая операция одновременно увеличивает один счёт (дебет) и уменьшает другой (кредит) на ту же сумму. Остаток по любому счёту = сумма всех дебетов минус сумма всех кредитов.

Связка «план счетов — регистр бухгалтерии» работает так же, как связка «справочник — регистр накопления», только с усложнённой логикой.

### Чем это отличается от регистра накопления

| | Регистр накопления | Регистр бухгалтерии |
|---|---|---|
| Запись | Приход или Расход | Дебет и Кредит одновременно |
| Измерения | Произвольные | Обязательно счёт + субконто |
| Остаток | По измерениям | По счёту и субконто |
| Требует | Ничего дополнительного | Плана счетов |

### Для «Канцтоваров» это не нужно

В «Канцтоварах» мы ведём простой складской учёт — регистр накопления справляется. Бухгалтерский учёт с двойной записью — отдельная, более сложная тема, которая выходит за рамки этой книги. Здесь важно знать, что такие объекты существуют, понимать их назначение и уметь опознать их в дереве реальной конфигурации.

Понимание того, зачем в дереве есть план счетов и регистр бухгалтерии, не равно пониманию методологии бухгалтерского учёта. Двойная запись, активно-пассивные счета, корреспонденция счетов — это отдельная большая область знаний со своими стандартами и правилами.

### Главное

План счетов — список адресов учёта; к каждому счёту привязаны субконто — дополнительные аналитические разрезы, уточняющие, *чего* и *где* учитывается сумма. Регистр бухгалтерии хранит проводки: каждая запись одновременно затрагивает два счёта (дебет и кредит). Это реализация принципа двойной записи. Связка «план счетов + регистр бухгалтерии» — аналог «справочник + регистр накопления», но для бухгалтерской логики.

### Контрольные вопросы

1. Чем план счетов похож на справочник — и чем отличается?
2. Что такое субконто и зачем оно нужно?
3. Чем проводка в регистре бухгалтерии отличается от движения в регистре накопления?
4. Почему для «Канцтоваров» не нужен регистр бухгалтерии?
5. Если у счёта нет субконто — какую информацию даёт остаток по этому счёту?

### Упражнения

**Найди в Конфигураторе.** Откройте демо-базу. Найдите план счетов в дереве метаданных. Откройте любой счёт и посмотрите, есть ли у него субконто. На какие справочники они ссылаются? Что это говорит о том, в каких разрезах ведётся учёт по этому счёту?

**Посмотри и ответь.** Найдите в демо-базе регистр бухгалтерии и откройте его вкладку «Регистраторы». Сколько документов пишут в него? Что произойдёт с бухгалтерскими остатками по счёту, если убрать один из документов-регистраторов из настроек движений — изменятся ли уже записанные проводки или только поведение будущих документов?

**Спроектируй.** Дана конфигурация с планом счетов, где есть счёт 62 «Расчёты с покупателями» без субконто. Бухгалтер жалуется: «Я вижу общий долг покупателей, но не вижу, кто именно должен». Какое субконто нужно добавить к счёту 62? На какой справочник оно должно ссылаться?

### Ответы и указания

*Вопрос 2.* Субконто — дополнительный аналитический разрез, привязанный к счёту. Без субконто остаток по счёту — это одна сумма. С субконто — детализированный учёт: остаток по счёту в разрезе каждого значения субконто (по каждому товару, по каждому складу, по каждому контрагенту).

*Вопрос 5.* Если субконто нет — остаток показывает только общую сумму по счёту без детализации. Например, «на счёте 51 — 500 000 рублей», но неизвестно, на каком именно расчётном счёте.

*Упражнение «Спроектируй».* К счёту 62 нужно добавить субконто с типом «СправочникСсылка.Контрагенты». После этого регистр будет хранить остаток не просто по счёту 62, а по счёту 62 в разрезе каждого контрагента — бухгалтер увидит долг отдельно по каждому покупателю.

### Мостик

Регистры бухгалтерии — для финансового учёта с двойной записью. Следующий объект тоже связан с учётом, но совсем другого рода: регистры расчёта хранят начисления — зарплату, отпускные, штрафы.
