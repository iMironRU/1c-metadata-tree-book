## § 0.3. Первый запуск Конфигуратора

### Открытие

Все следующие параграфы и модули этого курса предполагают, что Конфигуратор открыт рядом с книгой. Этот параграф — практический: запускаем 1С, открываем Конфигуратор и находим дерево метаданных.

### Что нужно заранее

Для работы по курсу вам понадобится:

- Установленная платформа 1С:Предприятие 8.3 (любая версия 8.3.х).
- Информационная база для работы — можно создать пустую или использовать демонстрационную базу, которая поставляется с платформой.

Если платформа ещё не установлена — установите её сейчас. Установщик запускается стандартно, никаких специальных настроек не требуется.

### Шаг 1: запуск

Запустите 1С:Предприятие. Появится окно запуска с двумя вариантами:

```
┌──────────────────────────────────────────┐
│  Запуск 1С:Предприятие                   │
│                                          │
│  [1С:Предприятие]   [Конфигуратор]       │
│                                          │
│  Информационная база:                    │
│  ○ Демонстрационная база                 │
└──────────────────────────────────────────┘
```

Выберите базу и нажмите «Конфигуратор».

### Шаг 2: интерфейс Конфигуратора

Конфигуратор откроется. Вы увидите окно, разделённое на части. Нас интересует левая панель — дерево конфигурации. Если оно не видно, откройте его через меню:

```
Конфигурация → Открыть конфигурацию
```

Или используйте сочетание клавиш F4.

### Шаг 3: первый взгляд на дерево

В левой панели — дерево. Вверху — корень с именем вашей конфигурации. Ниже — ветки. Раскройте несколько из них.

```
📁 Конфигурация (имя вашей базы)
│
├── 📁 Общие
├── 📁 Справочники
├── 📁 Документы
├── 📁 Журналы документов
├── 📁 Перечисления
├── 📁 Отчёты
├── 📁 Регистры сведений
├── 📁 Регистры накопления
└── ...
```

Ветки, которые нас интересуют прежде всего: «Справочники», «Документы», «Регистры сведений», «Регистры накопления». Именно с ними мы будем работать в курсе.

### Шаг 4: открыть объект

Раскройте ветку «Справочники». Найдите любой справочник. Дважды щёлкните по нему.

Откроется окно настройки объекта. Здесь несколько вкладок. Пока нас интересуют только «Основные» и «Данные» — с ними мы и будем работать. Остальные вкладки увидите, не пугайтесь — вернёмся к ним позже.

### Шаг 5: закрыть объект и вернуться к дереву

Закройте окно объекта. Вы снова видите дерево. Так будет устроена большая часть работы в курсе: выбрать объект в дереве → открыть → изучить или изменить → закрыть → вернуться к дереву.

### Главное

Конфигуратор запускается из стандартного окна выбора режима 1С. Дерево метаданных находится в левой панели и открывается через меню «Конфигурация → Открыть конфигурацию» или клавишей F4. Основные ветки — «Справочники», «Документы», «Регистры сведений», «Регистры накопления». Объект открывается двойным щелчком.

### Контрольные вопросы

1. Как открыть Конфигуратор, если у вас уже запущена 1С в пользовательском режиме?
2. Где находится дерево метаданных в интерфейсе Конфигуратора?
3. Как открыть отдельный объект метаданных для редактирования?
4. Какие ветки дерева мы будем изучать в первую очередь?
5. Что означает, если ветка «Справочники» пустая?

### Упражнения

**Сделай в Конфигураторе.** Запустите Конфигуратор. Откройте дерево метаданных. Раскройте все основные ветки по очереди. Подсчитайте: сколько объектов в каждой из веток «Справочники», «Документы», «Регистры накопления»? Запишите числа.

**Посмотри и ответь.** Откройте любой справочник двойным щелчком. Посмотрите на список вкладок. Найдите вкладку «Данные». Что там есть? Сколько реквизитов у этого справочника? Как вы думаете, что означает каждый из них?

**Спроектируй.** Не открывая никаких объектов — только глядя на названия веток и названия объектов в дереве — попробуйте угадать, чем занимается фирма, для которой сделана эта конфигурация. Запишите свою гипотезу. Мы вернёмся к ней в конце курса.

### Ответы и указания

*Вопрос 1.* Одновременно запустить и Конфигуратор, и пользовательский режим — можно, они работают независимо. Для этого запустите 1С второй раз и выберите «Конфигуратор» для той же базы.

*Вопрос 5.* Пустая ветка «Справочники» означает, что в конфигурации не создано ни одного справочника. Это типично для новой, только что созданной пустой конфигурации.

### Мостик

Конфигуратор открыт, дерево найдено. Теперь посмотрим на кое-что, что вы точно заметили, но мы пока не объяснили: у каждого объекта есть вкладка «Модуль». Что это такое — и почему мы туда не заходим.
