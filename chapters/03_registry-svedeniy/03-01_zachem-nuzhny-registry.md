# Модуль 3. Таблицы фактов — регистры сведений

---

## § 3.1. Зачем нужны регистры: хранить то, что нельзя вычислить из документов

### Открытие

В Модуле 2 мы создали документ «ПоступлениеТоваров». Он фиксирует факт: товар пришёл, контрагент такой-то, список позиций — вот. Документ — это событие, зафиксированное в момент, когда оно случилось.

Но теперь у главного бухгалтера «Канцтоваров» возникает вопрос: сколько ручек сейчас лежит на складе?

Откройте дерево метаданных в Конфигураторе. Мы уже знаем справочники и документы. Теперь — третья ветка.

```
Дерево метаданных
│
├── Справочники       ← существительные
├── Документы         ← глаголы
├── Регистры сведений ← мы здесь
├── Регистры накопления
└── ...
```

### Проблема: документы не предназначены для хранения состояния

Документы хранят события. Справочники хранят сущности. Но ни то ни другое не предназначено для хранения **текущего состояния** дел.

Какова сейчас цена ручки? Цена не вычисляется из событий — она просто меняется. Сегодня 12.50, с завтра — 14.00. Это не результат чего-то, что произошло в прошлом. Это актуальное состояние.

Курс доллара, ставка налога, контактное лицо у контрагента — всё это значения, которые верны на какой-то момент времени и могут меняться. Хранить их в документах — значит каждый раз пересчитывать. Держать в справочнике — терять историю.

> Документы сами по себе не предназначены для хранения текущего состояния — они фиксируют только факты прошлого. Для хранения состояний, которые меняются со временем, в 1С есть регистры.

Есть и особый вид состояния — то, которое нельзя просто записать: остатки товаров на складе. Остаток — это не фиксированное значение, а результат всех движений: сколько пришло, сколько ушло. Для таких данных существуют регистры накопления — мы разберём их в Модуле 4. Здесь нас интересует более простой случай: значения, которые просто хранятся и меняются.

### Два разных вопроса

Система задаёт два принципиально разных вопроса о данных:

**Вопрос о событии:** «Что произошло 15 марта?» — ответ в документе.

**Вопрос о состоянии:** «Какова цена ручки сегодня?» / «Каков сейчас курс доллара?» — ответ должен быть где-то ещё.

Для хранения состояния в 1С есть особые объекты — **регистры**. В этом модуле мы изучим самый простой из них: регистр сведений.

### Два вида регистров — два вида состояния

В дереве метаданных два основных вида регистров:

| | Что хранит | Пример |
|---|---|---|
| Регистр сведений | Значения свойств — «чему равно» | Цена товара = 12.50 |
| Регистр накопления | Суммируемые величины — «сколько стало» | Остаток ручек = 47 шт. |

В этом модуле — регистр сведений. Регистр накопления — в Модуле 4.

### Главное

Документы и справочники не предназначены для хранения текущего состояния. Документ — это запись события, справочник — запись сущности. Состояния, которые меняются со временем — цена, курс, ставка — это отдельный слой данных. Для него в 1С есть регистры сведений. Для состояний, которые вычисляются из движений (остатки), — регистры накопления (Модуль 4).

### Контрольные вопросы

1. Почему нельзя хранить текущую цену товара в документах?
2. Чем «вопрос о событии» отличается от «вопроса о состоянии»?
3. Чем регистр сведений отличается от регистра накопления — на уровне бизнес-смысла?
4. Почему регистры — отдельная ветка дерева метаданных, а не часть справочников или документов?
5. Назовите три примера данных, которые логично хранить в регистре сведений.

### Упражнения

**Найди в Конфигураторе.** Откройте ветку «Регистры сведений» в демо-базе. Сколько там регистров? Выберите три и попробуйте угадать по названию, что именно каждый хранит.

**Посмотри и ответь.** Откройте регистр сведений «КурсыВалют» (или аналогичный) в демо-базе. Посмотрите на его содержимое в пользовательском режиме. Как оно выглядит — как таблица? Что в строках, что в колонках? Чем это отличается от списка элементов справочника?

**Спроектируй.** Фирма «Канцтовары» хочет хранить следующие данные: текущие цены на товары, расписание работы складов (с какого по какой час), контактное лицо у каждого контрагента. Для каждого пункта решите: это документ, справочник или регистр сведений? Обоснуйте.

### Ответы и указания

*Вопрос 3.* Регистр сведений хранит значения, которые просто меняются: цена сегодня такая, завтра другая. Регистр накопления хранит то, что вычисляется из движений: сколько пришло минус сколько ушло — это остаток. Первый хранит «какое значение», второй — «сколько накопилось».

*Упражнение «Спроектируй».* Цены на товары — регистр сведений (меняются, нужна история). Расписание складов — регистр сведений (периодически обновляется). Контактное лицо у контрагента — скорее всего, реквизит или табличная часть справочника «Контрагенты» (это свойство сущности, а не самостоятельное изменяющееся состояние).
